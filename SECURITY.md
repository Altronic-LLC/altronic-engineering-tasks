# Security Policy

## Supported Versions

This project is a single-version web application. Security fixes are applied to the current `main` branch only.

| Version | Supported |
|---------|-----------|
| Latest (`main`) | Yes |
| Older deployments | No |

## Reporting a Vulnerability

**Please do not open a public GitHub issue for security vulnerabilities.**

To report a vulnerability privately, email:

**ray.white@altronic-llc.com**

Include the following in your report:

- Description of the vulnerability
- Affected file(s) and line numbers if known
- Steps to reproduce or proof-of-concept
- Potential impact / exploit scenario
- Any suggested remediation

You may share supporting files as a [secret GitHub Gist](https://gist.github.com) link included in your email.

You can expect an acknowledgement within **5 business days**. We will work with you to assess and address the issue before any public disclosure.

## Disclosure Policy

We follow a coordinated disclosure approach:

1. Reporter submits findings privately via email
2. Maintainers acknowledge receipt within 5 business days
3. Maintainers assess severity and plan remediation
4. Fix is developed and deployed
5. Reporter is notified when the fix is live
6. Public disclosure (if any) is agreed upon between reporter and maintainers

We ask that you give us reasonable time to address the issue before any public disclosure.
